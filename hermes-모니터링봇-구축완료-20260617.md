## 8. 구축 중 발견한 핵심 이슈 & 해결 (재현/트러블슈팅용)

| 문제 | 원인 | 해결 |
|------|------|------|
| 무료 모델이 도구를 안 부름 | `owl-alpha`(Nous stealth)는 tool 등록돼도 호출 안 함 | **Gemini 2.5 Flash로 교체** |
| 다 정상인데 "도구 없음"이라 답 | `get_active_alarms`가 ALARM 0건이라 빈 결과 → 모델 오해 | SOUL.md에 도구 사용 지침 명시 |
| CloudWatch 권한 거부 | `default` 프로파일이 S3 전용 키 | 전용 IAM `hermes-cloudwatch-ro` + `[hermes-cw]` 프로파일 신설 |
| gcloud 일부 명령 깨짐 | 설치 경로 공백 `...\Cloud SDK\...` | 정션 `C:\gsdk`로 공백 없는 경로 생성 |
| 새 GCP 프로젝트 생성 불가 | 회사 org 정책 (Project Creator 권한 없음) | bonseung 소유의 기존 프로젝트(`gen-lang-client-...`) 재사용 |
| **봇이 "응답하지 않음"** (최대 난관) | 앱이 **Workspace 부가기능** 모드 → 발행 SA가 `chat-api-push`가 아닌 **`service-55903507998@gcp-sa-gsuiteaddons...`** | **그 SA에 토픽 Publisher 권한 부여** → 즉시 해결 |

> 교훈: Google Chat 봇이 Pub/Sub로 메시지를 못 받으면, **Chat 앱 Configuration 화면의 "서비스 계정 이메일"** 을 확인해 그 SA에게 토픽 Publisher를 줘야 한다. (부가기능 모드면 `gsuiteaddons` SA)
