# 토스(Toss) 디자인 가이드 — 시각 언어 (디자인 인수인계용)

> ⚠️ **이 프로젝트의 디자인 SSOT는 이제 `docs/styleseed/`(StyleSeed)다.**
> 모든 UI는 StyleSeed를 따른다(루트 `CLAUDE.md` 강제 · 실무 토큰은 `docs/styleseed/PROJECT-RULES.md` 우선).
> 단일 액센트=토스블루 `#3182F6`는 유지하되, **중립·텍스트·상태·간격·라운드·그림자·모션 토큰은 StyleSeed 값을 따른다**(이 문서와 수치가 다르면 StyleSeed 우선). 이 파일은 토스 톤에 대한 보조 참고로만 남긴다.

> 이 문서는 **"어떻게 보이고 느껴져야 하는가"**(색·타이포·간격·모션·톤)만 규정한다.
> **무엇을 보여줄지(기능·데이터)와 화면 구성은 `FUNCTIONAL-SPEC.md` 및 디자이너의 자유 설계** 영역이다.
> 즉 여기 토큰/원칙은 **"스킨"**, 정보구조는 디자이너가 짠다.

## 0. 한 문장 철학

> **본질만 남기고 군더더기를 덜어낸, 신뢰감 있고 친근한 인터페이스.**

토스 디자인의 4대 감각:
1. **간결함(Simplicity)** — 한 화면 한 목적. 한 번에 하나의 결정만 묻는다.
2. **신뢰(Trust)** — 금융 수준의 또렷함. 숫자는 크고 정확하게, 상태는 색으로 분명하게.
3. **친근함(Friendliness)** — 부드러운 라운드, 따뜻한 회색, 사람 말투의 한국어 카피.
4. **여백(Breathing room)** — 빽빽하지 않다. 여백이 위계와 집중을 만든다.

## 1. 컬러 토큰

### 1.1 브랜드 / 액션
| 토큰 | HEX | 용도 |
|---|---|---|
| `blue/primary` | `#3182F6` | 토스 시그니처 블루. 주요 액션·강조·활성 상태·링크 |
| `blue/pressed` | `#1B64DA` | hover/press(진하게) |
| `blue/weak-bg` | `rgba(49,130,246,0.10)` | 선택/활성 항목의 옅은 블루 배경 |

> **블루는 아껴 쓴다.** "지금 누를 것 / 지금 활성 / 가장 중요한 숫자"에만. 남발하면 토스 느낌이 사라진다.

### 1.2 그레이스케일 (토스의 핵심 — 화면의 90%는 회색조)
| 토큰 | HEX | 일반 용도 |
|---|---|---|
| `white` | `#FFFFFF` | 표면(카드 등) |
| `grey/50` | `#F9FAFB` | 아주 옅은 표면 |
| `grey/100` | `#F2F4F6` | **페이지 배경**, 옅은 칩/트랙 |
| `grey/200` | `#E5E8EB` | 보더 헤어라인, 구분선 |
| `grey/300` | `#D1D6DB` | 비활성 보더 |
| `grey/400` | `#B0B8C1` | 플레이스홀더, 약한 아이콘 |
| `grey/500` | `#8B95A1` | 흐린 보조 텍스트 |
| `grey/600` | `#6B7684` | 보조 텍스트 |
| `grey/700` | `#4E5968` | 진한 보조 텍스트 |
| `grey/800` | `#333D4B` | 부제목 |
| `grey/900` | `#191F28` | **본문/제목(거의 검정)** |

### 1.3 의미색 (Semantic)
| 의미 | HEX | 옅은 배경(틴트) | 용도 |
|---|---|---|---|
| 긍정/정상 | `#15C47E` (민트그린) | `rgba(21,196,126,0.12)` | OK·상승양호·성공 |
| 부정/위험 | `#F04452` (레드) | `rgba(240,68,82,0.10)` | 에러·경보·임계 초과 |
| 주의 | `#FF9500` (오렌지) | `rgba(255,149,0,0.13)` | 경고·임박 |
| 정보 | `#3182F6` (블루) | `rgba(49,130,246,0.10)` | 중립 정보 |

> 상태는 **색 + 텍스트(또는 아이콘)** 를 함께 쓴다(색맹 대비). 색만으로 의미를 전하지 않는다.

### 1.4 라이트 우선
기본은 **라이트 테마**(밝은 회색 셸 + 흰 표면). 다크가 필요하면 같은 위계(배경<표면<텍스트 대비)를 어두운 쪽으로 반전하되, **라이트를 1순위**로 한다.

## 2. 타이포그래피

- **폰트:** 토스는 자체 *Toss Product Sans*를 쓴다. 구현 가능한 동등 대체는 **Pretendard**(무료, 한글 메트릭 거의 동일). 시스템 폴백: `-apple-system, "Apple SD Gothic Neo", "Malgun Gothic", system-ui, sans-serif`.
- **숫자:** 금융 UI의 핵심. **크고 굵게, `tabular-nums`(자리수 고정)**. 큰 지표 숫자는 글자 간격 살짝 좁게(`letter-spacing: -0.02em`).
- **한글 가독:** 행간 `line-height: 1.5~1.6`. 본문 14px 기준.

### 2.1 위계(권장 스케일 — 절대값 아닌 가이드)
| 역할 | 크기/굵기 | 색 |
|---|---|---|
| 큰 지표 숫자 | 28~40px / 700 | `grey/900` (또는 의미색) |
| 페이지 제목 | 22~26px / 700 | `grey/900` |
| 섹션/카드 제목 | 15~17px / 700 | `grey/900` |
| 본문 | 14px / 400~500 | `grey/900` |
| 보조 설명 | 12.5~13px / 400 | `grey/600` |
| 라벨/메타(작은 대문자) | 11~12px / 600, `letter-spacing: 0.04em` | `grey/500` |

> **큰 숫자 + 작은 설명**의 대비가 토스의 시그니처. 숫자를 주인공으로.

---

## 3. 간격 · 형태 · 깊이

### 3.1 스페이싱
- **8pt 그리드**(4의 배수, 주로 8·12·16·20·24).
- **넉넉하게.** 카드 내부 패딩 16~20px, 섹션 간 간격 16~24px. 빽빽함보다 여백.

### 3.2 라운드(border-radius)
| 요소 | 반경 |
|---|---|
| 카드/큰 표면 | 16~20px |
| 입력/작은 표면 | 12~14px |
| 버튼 | 12px 또는 pill(9999) |
| 칩/뱃지/토글 | pill(9999) |
| 아바타/원형 액션 | 50% |

> 토스는 **둥글다.** 날카로운 직각은 피한다. 단, 과하게 통통하지도 않게(16~20px가 기준).

### 3.3 보더 & 그림자
- **헤어라인 보더**(`grey/200`, 1px)로 표면을 가르되, 장식적 굵은 보더는 금지.
- **그림자는 아주 부드럽게**(낮은 불투명도, 넓은 blur). 떠 있는 느낌만 살짝.
  - 카드 기본: `0 1px 3px rgba(0,0,0,0.04), 0 4px 14px rgba(0,0,0,0.04)`
  - hover 강조: 블루 링 + 부드러운 리프트 `0 0 0 1px #3182F6, 0 10px 26px rgba(49,130,246,0.16)`
  - 팝오버/모달: `0 20px 60px rgba(0,0,0,0.16)`
- 보더와 그림자를 **둘 다 무겁게 쓰지 않는다.** 하나는 거의 보일 듯 말 듯.

---

## 4. 컴포넌트 원칙 (형태는 자유, 결만 통일)

- **카드:** 흰 표면 + 큰 라운드 + 헤어라인/부드러운 그림자. 그림자보다 **여백과 배경 대비**로 떠 보이게.
- **버튼(주):** 블루 채움 + 흰 글씨, pill 또는 12px 라운드. hover 시 `blue/pressed`.
- **버튼(보조)/토글:** 흰 배경 + 회색 보더 + 회색 글씨, 활성 시 블루 채움 또는 블루 텍스트.
- **칩/뱃지:** pill, 옅은 틴트 배경 + 의미색 텍스트(상태/카운트 표기에 적합).
- **상태 표시:** 작은 점(dot) 또는 pill + 텍스트, 의미색 사용.
- **입력:** 흰 배경 + 회색 보더, focus 시 **블루 보더 + 옅은 블루 글로우**(`0 0 0 3px rgba(49,130,246,0.15)`).
- **리스트:** 항목 간 헤어라인 또는 간격. 좌측 의미색 강조바로 심각도 표현 가능.
- **아이콘:** 얇은 라인 또는 단색 채움. 회색이 기본, 활성/강조만 블루.

---

## 5. 모션

- **짧고 부드럽게.** 전환 120~200ms, `ease`/`ease-out`. 과한 바운스·긴 애니메이션 금지.
- **마이크로 인터랙션은 절제해서, 의미 있는 순간에만**(hover 리프트, 로딩 스피너, 전송 상태, 탭 전환).
- **로딩:** 얇은 원형 스피너(블루 포인트) 또는 스켈레톤. 과한 연출 없이.
- **숫자 변화**가 중요하면 가벼운 count-up 정도까지는 OK(과용 금지).

---

## 6. 톤 & 보이스 (한국어 카피)

- **사람 말투, 짧고 분명하게.** "데이터 없음 — 수집 전이에요" 처럼 군더더기 없이 친근하게.
- **전문용어는 풀어준다.** 필요하면 작은 도움말(ⓘ)로 한 줄 설명(예: p95 = 가장 느린 5% 요청 기준 체감지연).
- **불안을 부추기지 않는다.** 정상일 땐 담백하게("모두 정상"), 문제일 땐 분명하되 과장 없이.
- 이모지는 **포인트로만** (남발 금지).

---

## 7. 안티패턴 (토스답지 않은 것 — 피하기)

- 보라색 그라데이션, 화려한 멀티컬러 배경 → ✗ (회색조 + 절제된 블루가 기본)
- 굵고 진한 장식 보더, 짙은 다중 그림자 → ✗ (헤어라인 + 부드러운 그림자)
- 정보 과밀(빽빽한 표/촘촘한 위젯) → ✗ (여백과 위계로 덜어내기)
- 작고 흐릿한 숫자 → ✗ (지표 숫자는 크고 또렷하게)
- 색만으로 상태 전달 → ✗ (색 + 텍스트/아이콘 병행)
- 제각각인 라운드/간격 → ✗ (8pt 그리드 + 16~20px 라운드로 통일)

## 8. 빠른 참조 — CSS 변수 스니펫(선택)

```css
:root {
  /* 배경/표면 */
  --bg: #f2f4f6; --surface: #ffffff; --surface-2: #f9fafb;
  /* 보더 */
  --border: #e5e8eb; --border-soft: #f2f4f6;
  /* 텍스트 */
  --text: #191f28; --text-sub: #6b7684; --text-weak: #8b95a1;
  /* 브랜드 */
  --blue: #3182f6; --blue-pressed: #1b64da; --blue-weak: rgba(49,130,246,.10);
  /* 의미색 */
  --positive: #15c47e; --negative: #f04452; --warning: #ff9500; --info: #3182f6;
  --positive-bg: rgba(21,196,126,.12); --negative-bg: rgba(240,68,82,.10); --warning-bg: rgba(255,149,0,.13);
  /* 깊이 */
  --shadow-card: 0 1px 3px rgba(0,0,0,.04), 0 4px 14px rgba(0,0,0,.04);
  --shadow-hover: 0 0 0 1px #3182f6, 0 10px 26px rgba(49,130,246,.16);
  --shadow-pop: 0 20px 60px rgba(0,0,0,.16);
  /* 라운드 */
  --r-card: 18px; --r-input: 12px; --r-pill: 9999px;
  /* 타이포 */
  --font: Pretendard, -apple-system, "Apple SD Gothic Neo", "Malgun Gothic", system-ui, sans-serif;
}
```

> 이 변수는 **출발점**일 뿐. 정확한 수치는 디자이너가 화면 설계에 맞춰 조정한다.
> 핵심은 토큰값이 아니라 **§0의 철학과 §1~§7의 결**이다.
